Authorising access to OSS: Notes for Information Custodians

As an Information Custodian, there are two parts that you play in arranging access:

  1. Signing and submitting 'single user' access request forms completed by individual staff users. Using the 'authorisation codes' form, you will newed to specify which areas (college or department(s)) the user is to have access to. Please note that before your signature can be accepted, you will need to have returned your 'Information Custodian authorisation of signature' form.
  2. Completing and submitting 'multi-user' access request forms on behalf of a group of staff users (note that group set-up only applies to read-only access).

OSS 'multi user' access forms:
 What this form is for
OSS User Change Multiple (285kb) To be used where an identical request is to be submitted for a large number of users requiring a read-only responsibility.
